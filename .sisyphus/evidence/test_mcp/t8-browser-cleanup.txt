# T8 Evidence: Browser Resource Cleanup

## Change Summary
Fixed send_once function in main.py to ensure playwright browser resources are properly cleaned up even when open_chat_page() throws an exception.

## Problem Identified
Original code had `p, context, page = await open_chat_page(config)` OUTSIDE the try block. If open_chat_page() threw an exception:
- We never entered the try block
- We never reached the finally block
- Browser resources leaked (playwright process, context)

## Changes Made

### 1. Initialize resources to None before try (lines 275-277)
```python
p = None
context = None
page = None
try:
    p, context, page = await open_chat_page(config)
```

### 2. Updated exception handler to check page is not None (lines 306-313)
```python
except Exception as exc:
    logger.debug(f"Send failed: {exc}")
    if page is not None:
        debug_path = await save_error_snapshot(page, exc)
        raise RuntimeError(f"Task failed. Debug file: {debug_path}") from exc
    raise
```

### 3. Updated finally block to check before cleanup (lines 314-317)
```python
finally:
    if context is not None:
        await context.close()
    if p is not None:
        await p.stop()
```

## Why This Change
- Ensures cleanup happens even when open_chat_page() fails early
- Prevents resource leaks (orphaned browser processes)
- Guards against None access in exception handler
- try/finally guarantees cleanup runs

## Verification
- If open_chat_page() throws, finally block still runs
- No AttributeError when trying to close() or stop() on None
- Debug snapshot only taken if page was actually created
