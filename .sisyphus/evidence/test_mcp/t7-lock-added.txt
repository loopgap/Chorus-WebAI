# T7 Evidence: Global State Lock Protection

## Change Summary
Added asyncio.Lock protection for LAST_INPUT global state in src/ui/handlers/events.py

## Changes Made

### 1. Added new lock for LAST_INPUT (lines 38-44)
```python
# Last input lock for thread-safe access
_LAST_INPUT_LOCK = None
def get_last_input_lock():
    global _LAST_INPUT_LOCK
    if _LAST_INPUT_LOCK is None:
        _LAST_INPUT_LOCK = asyncio.Lock()
    return _LAST_INPUT_LOCK
```

### 2. Updated _run_task to use lock (lines 413-415)
```python
async with get_last_input_lock():
    LAST_INPUT["template"] = template_label
    LAST_INPUT["content"] = raw_input
```

## Why This Change
- LAST_INPUT is a global dict accessed by multiple async handlers
- Without lock protection, concurrent access could cause race conditions
- When _run_task yields during async iteration, other handlers could read inconsistent state
- The lock ensures atomic read/write access to LAST_INPUT

## Verification
- Concurrent access to LAST_INPUT is now protected by asyncio.Lock
- Lock is lazily initialized (created on first access)
- Pattern follows existing get_login_lock() implementation
